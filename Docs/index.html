    Intro

    Caramel is a flexible system to create templates which are easy to maintain. This is the documentation where you can find every bit of necessary information to create templates and extend caramel.

    The system is build upon the same principles as an html dom. You have all possibilities to modify the dom or the nodes.

    Caramel already has some core plugins which handle the basic functionalities of a template engine. It also includes some predefined filter like markdown.

    Setup

    <?php
require_once "Caramel/Caramel.php";
?>

    Getting started with caramel is pretty easy. Just require the main class or use composer to require it.

    <?php
$caramel = new Caramel\Caramel();
?>

    After that create a new instance and you'r good to go.

    Syntax

    caramel markup

    html
  head
   title > test
  body
   div > hello world

    Caramel works with an indent style syntax. Each new line represents a new tag or function.

    The indent must be consistent throughout one template file.

    parsed output

    <html>
  <head>
    <title>test</title>
  </head>
  <body>
    <div>hello world</div>
  </body>
</html>

    This technique lets you save time while writing templates and puts the focus towards the result instead of the code.

    Since all templating functions are handled with plugins, all specific syntax definitions are documented there.

    Config

    The config can customize caramel to your needs.

    Caramel ships with a default config json file.

    {
  "extension":       "crml",
  "use_cache":       false,
  "cache_dir":       "../Cache",
  "comment_symbol":  "#",
  "show_comments":   true,
  "block_comments":  false,
  "variable_symbol": "@",
  "left_delimiter":  "{",
  "right_delimiter": "}",
  "file_header":     "Caramel template engine.",
  "self_closing":    [
    "br",
    ...
  ],
  "inline_elements": [
    "b",
    ...
  ]
}

    extension

    Sets the template file extension.

    <?php
$caramel->config()->set("extension","crml");
?>
    setting type default
    use_cache string “crml”

    use_cache

    Set if we want to use the cache or render the template on each site request.

    <?php
$caramel->config()->set("use_cache",true);
?>
    setting type default
    use_cache bool true

    cache_dir

    Sets where the cached templates will be saved.

    <?php
$caramel->config()->set("cache_dir","../Cache");
?>
    setting type default
    cache_dir string “../Cache”

    comment_symbol

    Sets the comment identifier.

    <?php
$caramel->config()->set("comment_symbol","#");
?>
    setting type default
    comment_symbol string “#”

    show_comments

    Sets if we want to see the comments in the html output.

    <?php
$caramel->config()->set("show_comments",true);
?>
    setting type default
    show_comments bool true

    block_comments

    Sets if we want to see the blocks as a comment in the html output.

    <?php
$caramel->config()->set("block_comments",false);
?>
    setting type default
    block_comments bool false

    variable_symbol

    Sets the variable identifier.

    <?php
$caramel->config()->set("variable_symbol","@");
?>
    setting type default
    variable_symbol string “@”

    left_delimiter

    Sets the left escape delimiter.

    <?php
$caramel->config()->set("left_delimiter","{");
?>
    setting type default
    left_delimiter string “{”

    right_delimiter

    Sets the right escape delimiter.

    <?php
$caramel->config()->set("right_delimiter","}");
?>
    setting type default
    right_delimiter string “}”

    file_header

    A comment which will be displayed at the first line of each template.

    <?php
$caramel->config()->set("file_header","Caramel template engine.");
?>
    setting type default
    file_header string “Caramel template engine.”

    self_closing

    Sets the default self closing html tags.

    <?php
$caramel->config()->set("self_closing", array("br", ...));
?>
    setting type default
    self_closing array array(“br”, …)

    inline_elements

    Sets the default html inline elements.

    <?php
$caramel->config()->set("inline_elements", array("b", ...));
?>
    setting type default
    inline_elements array array(“b”, …)

    Core

    Caramel is build around a Storage class which can hold any kind of data. All data models are implementing this Storage class to keep everything consistent.

    Core/Models/Storage

    set

    Sets a value to the Storage object.

    <?php
$Storage->set("path.to.my.value","hello world");
?>
    method params
    set string , mixed

    get

    Returns a value from the Storage object.

    <?php
$Storage->get("path.to.my.value");
?>
    method params
    get string

    merge

    Merges an array into the Storage object.

    <?php
$Storage->merge(array("key" => "value"));
?>
    method param
    merge array

    has

    Returns if a value exists in the Storage object.

    <?php
$has = $Storage->has("path.to.my.value");
?>
    method params
    has string

    delete

    Deletes a value from the Storage object.

    <?php
$Storage->delete("path.to.my.value");
?>
    method params
    delete string

    find

    Returns if a value exists in the Storage object.

    <?php
$found = $Storage->find("hello","world");
$found = $Storage->find(array("hello" => "world","goodbye" => "world"));
?>
    method params
    find array or string, string or null

    Core/Services/Service

    All services are implementing the Service class. The only purpose of that class is to serve the methods to implement other services into our current one.

    setCaramel

    Sets the Caramel service for the current service.

    <?php

?>

    setVars

    Sets the Vars service for the current service.

?>

    setConfig
